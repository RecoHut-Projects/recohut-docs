BPR

BPR stands for Bayesian Personalized Ranking. In matrix factorization (MF), to compute the prediction we have to multiply the user factors to the item factors:

x^ui=⟨wu,hi⟩=∑f=1kwuf⋅hif\hat{x}_{ui} = \langle w_u,h_i \rangle = \sum_{f=1}^k w_{uf} \cdot h_{if}x^ui​=⟨wu​,hi​⟩=f=1∑k​wuf​⋅hif​

The usual approach for item recommenders is to predict a personalized score x^ui\hat{x}_{ui}x^ui​ for an item that reflects the preference of the user for the item. Then the items are ranked by sorting them according to that score. Machine learning approaches are typically fit by using observed items as a positive sample and missing ones for the negative class. A perfect model would thus be useless, as it would classify as negative (non-interesting) all the items that were non-observed at training time. The only reason why such methods work is regularization.

BPR use a different approach. The training dataset is composed by triplets (u,i,j)(u,i,j)(u,i,j) representing that user uuu is assumed to prefer iii over jjj. For an implicit dataset this means that uuu observed iii but not jjj:

DS:={(u,i,j)∣i∈Iu+∧j∈I∖Iu+}D_S := \{(u,i,j) \mid i \in I_u^+ \wedge j \in I \setminus I_u^+\}DS​:={(u,i,j)∣i∈Iu+​∧j∈I∖Iu+​}

A machine learning model can be represented by a parameter vector ΘΘΘ which is found at fitting time. BPR wants to find the parameter vector that is most probable given the desired, but latent, preference structure >u>_u>u​:

p(Θ∣>u)∝p(>u∣Θ)p(Θ)∏u∈Up(>u∣Θ)=⋯=∏(u,i,j)∈DSp(i>uj∣Θ)\begin{align} p(\Theta \mid >_u) \propto p(>_u \mid \Theta)p(\Theta) \\ \prod_{u\in U} p(>_u \mid \Theta) = \dots = \prod_{(u,i,j) \in D_S} p(i >_u j \mid \Theta) \end{align}p(Θ∣>u​)∝p(>u​∣Θ)p(Θ)u∈U∏​p(>u​∣Θ)=⋯=(u,i,j)∈DS​∏​p(i>u​j∣Θ)​​

The probability that a user really prefers item iii to item jjj is defined as:

p(i>uj∣Θ):=σ(x^uij(Θ))\begin{align} p(i >_u j \mid \Theta) := \sigma(\hat{x}_{uij}(\Theta)) \end{align}p(i>u​j∣Θ):=σ(x^uij​(Θ))​​

Where σσσ represent the logistic sigmoid and x^uij(Θ)\hat{x}_{uij}(Θ)x^uij​(Θ) is an arbitrary real-valued function of ΘΘΘ (the output of your arbitrary model).

User u1u_1u1​ has interacted with item i2i_2i2​ but not item i1i_1i1​, so we assume that this user prefers item i2i_2i2​ over i1i_1i1​. For items that the user have both interacted with, we cannot infer any preference. The same is true for two items that a user has not interacted yet (e.g. item i1i_1i1​ and i4i_4i4​ for user u1u_1u1​).

To complete the Bayesian setting, we define a prior density for the parameters:

p(Θ)∼N(0,ΣΘ)p(\Theta) \sim N(0, \Sigma_\Theta)p(Θ)∼N(0,ΣΘ​)

And we can now formulate the maximum posterior estimator:

BPR−OPT=log⁡p(Θ∣>u)=log⁡p(>u∣Θ)p(Θ)=log⁡∏(u,i,j)∈DSσ(x^uij)p(Θ)=∑(u,i,j)∈DSlog⁡σ(x^uij)+log⁡p(Θ)=∑(u,i,j)∈DSlog⁡σ(x^uij)−λΘ∣∣Θ∣∣2\begin{equation} \begin{split} BPR-OPT &=\log p(\Theta \mid >_u)\\ &=\log p(>_u \mid \Theta) p(\Theta) \\ &= \log \prod_{(u,i,j) \in D_S} \sigma(\hat{x}_{uij})p(\Theta) \\ &= \sum_{(u,i,j) \in D_S} \log \sigma(\hat{x}_{uij}) + \log p(\Theta) \\ &= \sum_{(u,i,j) \in D_S} \log \sigma(\hat{x}_{uij}) - \lambda_\Theta ||\Theta||^2 \end{split} \end{equation}BPR−OPT​=logp(Θ∣>u​)=logp(>u​∣Θ)p(Θ)=log(u,i,j)∈DS​∏​σ(x^uij​)p(Θ)=(u,i,j)∈DS​∑​logσ(x^uij​)+logp(Θ)=(u,i,j)∈DS​∑​logσ(x^uij​)−λΘ​∣∣Θ∣∣2​​​

Where λΘλ_ΘλΘ​ are model specific regularization parameters.

Once obtained the log-likelihood, we need to maximize it in order to find our optimal ΘΘΘ. As the criterion is differentiable, gradient descent algorithms are an obvious choiche for maximization.

The basic version of gradient descent consists in evaluating the gradient using all the available samples and then perform a single update. The problem with this is, in our case, that our training dataset is very skewed. Suppose an item iii is very popular. Then we have many terms of the form x^uij\hat{x}_{uij}x^uij​ in the loss because for many users uuu the item iii is compared against all negative items jjj. The other popular approach is stochastic gradient descent, where for each training sample an update is performed. This is a better approach, but the order in which the samples are traversed is crucial. To solve this issue BPR uses a stochastic gradient descent algorithm that chooses the triples randomly.

The gradient of BPR-OPT with respect to the model parameters is:

∂BPR−OPT∂Θ=∑(u,i,j)∈DS∂∂Θlog⁡σ(x^uij)−λΘ∂∂Θ∣∣Θ∣∣2=∑(u,i,j)∈DS−e−x^uij1+e−x^uij∂∂Θx^uij−λΘΘ\begin{equation} \begin{split} \frac{\partial BPR-OPT}{\partial \Theta} &= \sum_{(u,i,j) \in D_S} \frac{\partial}{\partial \Theta} \log \sigma (\hat{x}_{uij}) - \lambda_\Theta \frac{\partial}{\partial\Theta} || \Theta ||^2\\ &= \sum_{(u,i,j) \in D_S} \frac{-e^{-\hat{x}_{uij}}}{1+e^{-\hat{x}_{uij}}} \frac{\partial}{\partial \Theta}\hat{x}_{uij} - \lambda_\Theta \Theta \end{split} \end{equation}∂Θ∂BPR−OPT​​=(u,i,j)∈DS​∑​∂Θ∂​logσ(x^uij​)−λΘ​∂Θ∂​∣∣Θ∣∣2=(u,i,j)∈DS​∑​1+e−x^uij​−e−x^uij​​∂Θ∂​x^uij​−λΘ​Θ​​​

In order to practically apply this learning schema to an existing algorithm, we first split the real valued preference term: x^uij:=x^ui−x^uj\hat{x}_{uij} := \hat{x}_{ui} − \hat{x}_{uj}x^uij​:=x^ui​−x^uj​. And now we can apply any standard collaborative filtering model that predicts x^ui\hat{x}_{ui}x^ui​.

The problem of predicting x^ui\hat{x}_{ui}x^ui​ can be seen as the task of estimating a matrix X:U×IX:U×IX:U×I. With matrix factorization the target matrix XXX is approximated by the matrix product of two low-rank matrices W:∣U∣×kW:|U|×kW:∣U∣×k and H:∣I∣×kH:|I|×kH:∣I∣×k:

X:=WHtX := WH^tX:=WHt

The prediction formula can also be written as:

x^ui=⟨wu,hi⟩=∑f=1kwuf⋅hif\hat{x}_{ui} = \langle w_u,h_i \rangle = \sum_{f=1}^k w_{uf} \cdot h_{if}x^ui​=⟨wu​,hi​⟩=f=1∑k​wuf​⋅hif​

Besides the dot product ⟨⋅,⋅⟩, in general any kernel can be used.

We can now specify the derivatives:

∂∂θx^uij={(hif−hjf) if θ=wuf,wuf if θ=hif,−wuf if θ=hjf,0 else \frac{\partial}{\partial \theta} \hat{x}_{uij} = \begin{cases} (h_{if} - h_{jf}) \text{ if } \theta=w_{uf}, \\ w_{uf} \text{ if } \theta = h_{if}, \\ -w_{uf} \text{ if } \theta = h_{jf}, \\ 0 \text{ else } \end{cases}∂θ∂​x^uij​=⎩⎨⎧​(hif​−hjf​) if θ=wuf​,wuf​ if θ=hif​,−wuf​ if θ=hjf​,0 else ​

Which basically means: user uuu prefer iii over jjj, let's do the following:

  • Increase the relevance (according to uuu) of features belonging to iii but not to jjj and vice-versa
  • Increase the relevance of features assigned to iii
  • Decrease the relevance of features assigned to jjj
